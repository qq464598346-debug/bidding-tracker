## 🚀 快速开始

### 1. 安装依赖

```bash
cd spider
pip install -r requirements.txt

# 安装Playwright浏览器（用于处理反爬网站）
playwright install chromium
```

### 2. 配置环境变量

```bash
cp .env.example .env
# 编辑 .env 文件，配置API密钥等参数（可选）
```

### 3. 启动系统

```bash
# 方式一：完整模式（采集 + 定时任务 + API服务）
python main.py

# 方式二：仅执行一次采集
python main.py --crawl

# 方式三：仅启动API服务
python main.py --api

# 自定义端口
python main.py --port 9000
```

启动后访问:
- **前端网站**: 直接打开 `index.html`
- **API文档**: http://localhost:8765/
- **API接口**: http://localhost:8765/api/bidding

## 🔧 核心功能

### 1. 多策略爬虫引擎
```
请求流程: API接口调用 → HTML页面抓取 → Playwright浏览器自动化 → 模拟备用数据
```
- **三级降级机制**: 每个数据源都有多套采集方案，确保数据可用性
- **智能反爬**: 随机UA、请求延迟、浏览器指纹伪装
- **错误重试**: 自动重试 + 指数退避

### 2. AI智能分析引擎
- ✅ **规则引擎版**: 无需任何API Key即可使用（默认）
- ✅ **OpenAI增强版**: 配置Key后自动启用，精度更高

分析能力:
- 🏷️ 自动分类（基础软件/解决方案/服务器/服务）
- 📊 相关度评分（0-1分，判断是否为目标领域）
- 🏷️ 智能标签生成
- 📝 关键信息提取（预算金额、地区、要求等）

### 3. 统一RESTful API

```bash
# 获取招标列表
GET /api/bidding?keyword=服务器&source=chinamobile&category=server&limit=20&sort=publish_time&order=desc

# 获取统计概览
GET /api/stats

# 手动触发采集
POST /api/crawl

# 查看数据源状态
GET /api/sources

# 获取分类定义
GET /api/categories
```

### 4. 定时采集任务
- 可配置采集间隔（默认60分钟）
- 后台自动运行，无需手动干预
- 采集日志记录到数据库

## ⚠️ 重要说明

### 关于真实数据采集
本爬虫系统提供了完整的框架和多种采集策略。但由于各运营商采购网站存在以下情况：

1. **反爬机制**: 部分网站有debugger检测、验证码等防护
2. **动态渲染**: 数据通过JavaScript异步加载
3. **接口变更**: API端点可能随时间变化

因此系统中内置了**模拟演示数据**作为备选方案，确保：
- 开发调试时有数据可用
- 前端界面可正常展示
- 功能逻辑可完整测试

### 生产环境部署建议
1. **使用代理IP池**: 避免单IP被封锁
2. **分布式部署**: 多节点并行采集
3. **对接付费API**: 如企查查、知了标讯等第三方数据接口
4. **增加缓存层**: Redis缓存热点数据
5. **消息队列**: Celery+Redis处理异步采集任务

## 🔄 切换数据模式

在前端 `index.html` 中修改 `CONFIG`:

```javascript
const CONFIG = {
    API_BASE: 'http://localhost:8765',
    USE_API: true,   // true = 从后端API获取数据, false = 使用本地模拟数据
};
```
